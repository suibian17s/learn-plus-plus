# README 用户向重写记录

## 背景

准备将 learn++ v1.1 开源到 GitHub，需要 README 更像标准开源项目首页，而不是只面向开发者的简短说明。

## 调整思路

- 开头使用大尺寸 learn++ logo 和居中标题。
- 增加版本、平台、许可证、技术栈 badge。
- 面向用户解释 learn++ 是什么、适合谁、能解决什么问题。
- 按模块介绍公告、课件、作业、讨论、多账号、后台运行和甘蔗 tutor。
- 补充安装、使用流程、隐私与安全、开发命令、技术栈、免责声明。

## 结果

README 现在更适合作为 GitHub 仓库首页，用户无需了解代码即可快速理解项目价值、安装方式和使用边界。

## 后续微调

- README 顶部大图标路径改为 `./resources/icon.png`，明确指向 GitHub 仓库源码中的图标资源。
- 隐私与安全部分补充 Windows 用户数据目录说明，解释免安装版和重装后仍保留账号/设置的原因。
- 免责声明中补充本项目主要使用 Claude Code 与 Codex 辅助开发。
