# 讨论区修复记录

## 背景

讨论区列表可以加载，但进入帖子后页面排版混乱，头像无法加载，点赞、评论、回复等交互也不能正常工作。之前做过静态 HTML 抓取优化，但只能改善“看内容”，不能让原站交互真正运行。

## 主要原因

讨论帖不是单纯的文本内容页。头像、点赞、评论、回复、楼层交互等功能依赖网络学堂原始页面的脚本、样式、Cookie、相对资源路径和浏览器环境。

把原网页 HTML 抓取后塞进 React 组件，会丢失这些运行条件，所以即使正文能显示，内部按钮也只是静态元素，头像和图片也容易因为登录态或路径问题加载失败。

## 修复思路

1. 讨论列表仍由 learn++ 自己展示，保持课程内导航体验。
2. 点击具体帖子时，不再进入静态抓取详情页，而是打开一个 learn++ 内置的 Electron 窗口加载网络学堂原始帖子页面。
3. 打开窗口前，把 API 会话中的登录 Cookie 同步到 Electron 默认浏览器会话，让原始页面具备登录态。
4. 新窗口禁用 Node 集成、启用隔离和沙箱，只承载网络学堂页面本身；外链仍交给系统浏览器处理。

## 当前结果

讨论帖会在 learn++ 内部完整页面中打开，原站样式、头像、点赞、评论、回复等功能由网络学堂自身脚本接管。旧的静态详情页保留为兜底入口，但提示用户打开完整讨论页。

## 后续注意

如果未来要把讨论区交互完全做成 learn++ 原生组件，需要进一步逆向网络学堂讨论区的点赞、回复、评论接口。当前方案优先保证真实可用，避免继续维护不完整的静态抓取页面。
