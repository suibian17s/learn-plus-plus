# 1.1.1 运行态刷新与后台恢复修复

本次修复围绕两个运行态问题展开：登录后首页偶发无法加载课程，以及程序在后台常驻一段时间后重新打开出现白屏。

处理思路：

- 课程加载不再只依赖网络学堂返回的“当前学期”。现在会先尝试当前学期，如果课程列表为空或接口短暂异常，会继续按学期列表兜底查找，找到可用课程后自动进入课程首页。
- 新增全局刷新入口，放在主界面右上角。点击后会刷新账号、课程列表和当前页面数据，适合网络状态变化、长时间后台恢复后手动同步。
- 后台窗口重新打开时，主进程会检查渲染进程状态；如果窗口长期隐藏、页面地址异常或渲染进程崩溃，会自动重新加载界面。
- 主进程新增恢复通知，前端收到后会校验登录状态并刷新当前缓存，减少后台常驻后的空白或过期状态。
- 版本号统一提升到 `1.1.1`，README、CLAUDE 交接文档与程序内关于页保持一致。

结果：

登录后的课程加载更稳，后台唤醒时会自动恢复界面状态，同时用户也可以通过“刷新”按钮主动同步全局数据。
