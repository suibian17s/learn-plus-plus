# 下载状态显示优化记录

## 背景

课件、作业附件和公告附件此前只在当前页面会话里知道“刚刚下载过”。关闭程序、切换页面或文件已经存在于下载目录时，按钮仍然显示“下载”，容易重复保存同一份文件。

## 修复思路

- 在主进程新增下载状态查询能力：根据当前下载目录和规范化文件名判断目标文件是否已经存在。
- 渲染进程同时参考两类信息：下载目录中的实际文件、持久化下载历史记录。
- 课件列表、作业详情附件、公告附件统一使用“下载 / 已下载”状态。
- 下载成功后把结果写入下载历史，并立即刷新按钮状态。

## 结果

- 文件已经在下载目录中时显示“已下载”。
- 历史记录中已有完成记录时也显示“已下载”。
- 点击“已下载”会打开所在文件夹，而不是再次下载。
- 课件上传时间、公告发布时间等展示为 `YYYY-MM-DD HH:mm:ss`。
