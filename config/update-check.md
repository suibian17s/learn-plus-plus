# 更新检测功能

本次新增自动与手动检测更新能力，版本号保持不变，README 不记录该项变更。

## 实现思路

- 主进程新增 `app:check-updates` 接口，访问 GitHub Releases 最新版本信息，并和当前程序版本比较。
- 主界面加载或从后台唤醒时会自动检测更新。
- 若发现新版本，会在窗口底部显示提示，并提供跳转到 GitHub Releases 的入口。
- 若检测失败，会在窗口底部显示错误提示，不使用弹窗。
- 自动检测失败次数按天保存在本地；同一天连续两次失败后，当天不再自动检测，避免频繁打扰用户。
- 关于页面新增 GitHub 项目地址和“手动检测更新”按钮，手动检测不受自动检测失败次数限制。

## GitHub 地址

https://github.com/suibian17s/learn-plus-plus

## 结果

用户打开主界面时可以自动得知是否有新版；如果网络或 GitHub 访问失败，会收到轻量提示，并在连续失败后自动降噪。用户也可以在“关于 learn++”里手动检查更新。
