# 甘蔗 tutor API 配置优化记录

## 背景

甘蔗 tutor 支持多个大模型服务商后，原先仍使用全局单个 API Key。用户在 Anthropic、OpenAI、DeepSeek、通义千问等服务之间切换时，容易出现“界面看似保存成功，但实际调用用了另一个服务商 Key”的问题。

## 参考

本轮参考了 GitHub 项目 `farion1231/cc-switch` 的配置切换思路：把模型服务看成可切换的配置档案，当前档案拥有自己的服务、模型、端点和密钥状态。

## 修复思路

- API Key 改为按服务商加密保存。
- 保留旧版本单 Key 的兼容读取，避免升级后丢失原有配置。
- 预设服务商只展示固定 Endpoint，减少误填风险。
- 自定义接口才允许编辑 Endpoint 和接口格式。
- 设置页切换服务商时，会立即显示该服务商是否已经保存 Key。

## 结果

- 每个服务商可以独立保存 Key。
- 切换模型服务后不会复用错误 Key。
- 甘蔗 tutor 调用接口时按当前服务商读取对应 Key 和 Endpoint。
