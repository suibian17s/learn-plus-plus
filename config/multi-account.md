# 多账号切换优化记录

## 背景

上一版“切换账号”本质上只是退出当前账号，再让用户重新登录。这不符合多账号使用场景：用户希望可以保存多个网络学堂账号，并在左上角菜单中直接切换。

## 修复思路

- 官方网页登录成功后，读取当前用户信息，并把当前 API 会话 cookies 保存为一个账号档案。
- 新增账号列表、添加账号、切换账号 IPC。
- 添加账号时会打开清华网络学堂官方登录窗口，登录成功后自动保存并切换到新账号。
- 切换账号时恢复该账号保存的 cookies，重新初始化网络学堂 API 会话，然后刷新课程列表。
- 兼容已有用户：打开账号菜单时会尝试把当前登录态迁移为账号档案。
- 登录页移除账号密码登录，只保留官方页面登录，避免绕过 WebVPN、二级验证等官方流程。

## 结果

- 左上角 logo 菜单会显示已保存账号列表。
- 可以直接在多个账号之间切换，不需要退出后重新登录。
- 可以从同一菜单继续添加新账号。
- 登录页只保留“通过清华网络学堂官方页面登录”。
