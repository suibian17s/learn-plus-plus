# Learn++ 2.0 全功能实现设计

## 范围

基于 `unsolved.md` 和 `config/uiupdate.md` 中记录的 2.0 UI 框架，完成所有遗留功能的真实业务实现。

## 项目分解

### P1: 数据与统计引擎（基础层）

**stats.ts — 学习统计服务**
- 监听 BrowserWindow show/hide，累积窗口可见时长
- 按天存储到 `stats.json`（`%APPDATA%/learn++/stats.json`）
- 连续天数：当天 ≥30min 算一天
- 课程进度：只看作业（已提交 / 总数），不含课件
- 今日重点三级优先级：
  - P0 紧急：当天 23:59 截止的作业
  - P1 重要：近 3 天截止的作业
  - P2 参考：新公告、新讨论
- 最近更新聚合各课程最新公告/课件/作业/讨论时间线
- IPC: `stats:getDashboard` 返回完整 Dashboard 数据

**search-index.ts — 全局搜索**
- 课程加载后自动构建反向索引，中文按字符 n-gram 分词
- 支持类型过滤：课程/作业/课件/公告/讨论/邮件
- 搜索结果携带跳转信息（type + courseId + targetTab + targetId）
- IPC: `search:query(q, typeFilter?)`
- 数据源由各模块主动 push 索引（indexNotices/indexFiles/indexHomeworks）

**学期切换**
- Settings 页增加学期下拉框，复用现有 `course:semesters` + `course:list` IPC
- 选择持久化到 settings store

**涉及文件**
- 新增：`src/main/services/stats.ts`、`src/main/services/search-index.ts`、`src/main/ipc/stats.ts`
- 修改：`src/main/index.ts`、`src/preload/index.ts`、`src/preload/api.d.ts`、`src/renderer/src/env.d.ts`
- 修改：`src/renderer/src/pages/Dashboard.tsx`、`src/renderer/src/pages/Settings.tsx`、`src/renderer/src/components/AppShell.tsx`

### P3: 甘蔗 Tutor 全功能

**角色定位**

甘蔗是 learn++ 全局式 AI 学习助手，可爱二次元正太形象。从左侧导航点击"甘蔗 Tutor"进入对话页面，用户通过自然语言与甘蔗对话，甘蔗可以直接操控本程序的各项能力（总结邮件、完成作业、搜索课件等）。对话风格可爱生动，略带俏皮但不失专业。

**对话风格**

支持两种风格，首次使用甘蔗对话时弹出风格选择（可爱风 / 正经风），默认可爱风。设置页也可随时更改。

- 可爱风：正太语气（"诶嘿～"、"交给我吧！"、"这个我来帮你搞定～"），学术内容准确但语气轻松，输出标注"仅供学习参考哦～"
- 正经风：专业学术助手语气，简洁直接，无卖萌用语，输出标注"仅供学习参考"

风格选择持久化到 settings store，注入 system prompt。

**3a. 全局 Agent 工具调用架构**

甘蔗 Tutor 是一个有工具调用能力的 AI agent。对话流中的 function calling 循环：

```
用户自然语言输入
  → AI 判断是否需要调用工具（function calling）
    → 需要：执行工具调用（查邮件/搜课件/扫作业等），结果注入对话上下文
    → AI 基于工具结果生成回复（流式输出）
```

工具注册表（初始版）：

| 工具名 | 功能 | 对应 IPC |
|--------|------|----------|
| `list_courses` | 列出所有课程 | course:list |
| `list_homeworks` | 列出课程作业 | hw:list |
| `list_emails` | 列出邮件（指定文件夹） | mail:list |
| `search_emails` | 搜索邮件 | mail:search |
| `get_email` | 获取邮件详情 | mail:get |
| `list_files` | 列出课程课件 | files:list |
| `list_notices` | 列出课程公告 | notice:list |
| `list_discussions` | 列出课程讨论 | disc:list |
| `summarize_content` | 总结指定内容 | tutorSummary |
| `complete_homework` | 一键完成作业（进入完整流程） | homeworkAutoComplete |
| `search_global` | 全局搜索 | search:query |
| `get_stats` | 获取学习统计 | stats:getDashboard |

工具调用走主进程 `src/main/services/tutor-agent.ts`（新增），编排 function calling → IPC 执行 → 结果回注对话。

**3b. 对话流**
- Tutor.tsx 输入框 → IPC → ai.ts 流式 + 工具调用 → renderer 逐字渲染
- 复用现有 `hwai:generate-chunk` / `hwai:generate-end` 流式通道
- 对话历史存 Zustand 内存（会话内保留，关窗口清空）
- 每条 assistant 消息底部标注"仅供学习参考哦～"
- Tutor 页右侧窄区域：甘蔗形象、在线状态、可用功能列表

**3c. 上下文管理**
- 已选中课程时自动在 system prompt 注入课程名+教师
- 最近 N 轮对话作为上下文窗口（含工具调用结果）
- Tutor 页提供"新对话"按钮清空上下文

**3d. 总结功能接入**
- 课件详情/公告/讨论页的甘蔗 Tutor 按钮 → 弹窗/抽屉展示总结结果
- 不跳转 Tutor 页
- 复用 tutorSummary IPC，传入 courseId + kind + 具体 item 内容

**3e. 快捷功能**
- 解释概念 / 生成复习计划 / 出题练习 / 知识总结 / 全局辅助 / 学习计划
- 每个按钮 = 预设 system prompt + 模板化 user prompt
- 统一走对话流通道

**3f. 一键完成作业**

初始版策略：先实现覆盖常见情况的通用方案，后续根据真实课程做专项优化。

交付格式判断：默认生成 DOCX。AI 在作业要求解析后自动判断是否应切换 PDF（判断依据：作业明确要求 PDF、涉及数学公式/代码排版、要求打印提交等）。判断规则封装为甘蔗 Tutor system prompt 中的格式判断 skill。

上下文收集（生成前自动执行）：
1. 读取作业描述文本，解析格式/字数/引用要求，同时判断交付格式
2. 检测老师附件（作业描述中附带的文件），自动下载并解析
3. 检测作业描述中引用的课程课件（"参考第X章"、"见课件PPT"等），获取对应内容
4. 以上全部作为 AI 生成上下文

生成流程：
- 从 Tutor 页进入（对话中请求"帮我完成XX作业"），需先选课程
- 学术诚信弹窗（浅黄底色，非红色）
- 显示上下文收集进度（扫描 → 解析附件 → 关联课件 → 格式判断）
- 生成参考答案（流式输出）→ 生成 DOCX/PDF 附件
- 全程可取消（AbortController），不自动提交
- 学术诚信确认状态持久化

后续优化策略：用户根据真实使用反馈，针对特定课程做专项适配。

### P4: 邮箱真实接入

**登录**
- `mail:login` IPC → BrowserWindow 打开 `mails.tsinghua.edu.cn`
- 用户完成认证后抓取 Cookie 存入 mail session
- 与 `browser-login.ts` 架构一致

**左侧导航交互**
- 点击 AppShell 中"邮箱"导航项 → 下方非线性动画展开子菜单（收件箱 / 草稿箱 / 已发送 / 已删除）
- 直接点击"邮箱"（不选子项）→ 默认进入收件箱

**数据获取**
- `mail:list(folder, page)` → HTTP 请求 webmail 对应页面，解析邮件列表
- `mail:get(mailId)` → 获取邮件正文
- `mail:star(mailId, starred)` → 星标操作
- `mail:delete(mailId)` → 删除/归档操作
- 列表缓存 5 分钟，正文按需加载后缓存 30 分钟

**UI**
- 复用现有 Mailbox.tsx 设计
- 搜索、筛选、排序在 renderer 侧从缓存列表过滤完成
- 星标、删除等操作在 learn++ 内完成（IPC → HTTP 请求 webmail）

**涉及文件**
- 新增：`src/main/services/mail-service.ts`、`src/main/ipc/mail.ts`
- 修改：`src/preload/index.ts`、`src/preload/api.d.ts`、`src/renderer/src/env.d.ts`

## 执行顺序

```
第1轮: P1 数据引擎
       ↓
第2轮: P2 ‖ P3 ‖ P4 三路并行
       ↓
第3轮: P5 UI 收尾
```

- P2 内部可并行：2a(筛选排序) + 2b(预览) → 2c(批量下载先依赖预览 IPC)
- P3 内部顺序：3a(对话流) → 3b(上下文) → 3c(总结) → 3d(快捷) + 3e(一键作业)
- P4 内部顺序：mail-service → Mailbox UI → 导航子菜单
- P5 在所有功能完成后做，避免冲突
